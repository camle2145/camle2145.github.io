Fantasy Football Stats

Fantasy Football Stats is a full-stack application for accessing NFL fantasy information.

Development

Firstly, I needed data for my database. Originally I used some CSV files available on GitHub to put into my database, but the information was outdated and not relevant at all to what I wanted to achieve. Later on, I used an external API to get data in JSON format, and used scripts to remove irrelvant information (such as defensive players, which aren't relevant in fantasy football), and to convert it into CSV files.

From there, I created a PostgresQL database using the CSV files. It took a few goes to ensure that columns were matching up and that keys were working, and some adjustments were required later on.

I used the Spring Boot framework to build my Java back-end, and although tutorials and guides helped, a lot of the learning I did was just through trial and error and running into issues as I went.

The front-end wasn't supposed to be anything fancy, so I made a simple front-end with the help of AI, as the front-end was simply a vehicle to display the learning I did on the back-end.

Languages/Tools Used

  • ☕Java
  • 🍃Spring Boot
  • 🐘PostgreSQL
  • 🟨Javascript
  • 🟦HTML/🟥CSS

ANDIE

ANDIE is a non-destructive image editor, written in Java.

Development

ANDIE was a university group project, meant to improve collaboration and development skills. It was a great introduction to Git and the dynamic of developing software in a team.

The project had two deadlines with a variety of requirements. A few features that I worked on were Gaussian Blur, area transparency, and CI/CD.

Gaussian Blur was the first feature I worked on, and it took a while. Familiarising myself with ANDIE's skeleton codebase took some learning, and Gaussian Blur had some relatively tricky maths that introduced some complications.

I developed area transparency as an extra feature, and I really enjoyed implenting it, using breadth-first search (BFS) to make similar colours transparent. It felt like a feature that a real image editor would have, and it was satisfying utilisng BFS, which I had learnt in another class.

CI/CD was straightforward to implement for ANDIE, with a few simple tests, packaging the project into a zip, and uploading it to a website for download. It wasn't a topic that I had explored extensively before, so it was really interesting exploring DevOps after spending most of the project writing code.
